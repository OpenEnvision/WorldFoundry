终端界面 (TUI) 在你的终端中提供了一个基于文本的交互式面板。这是浏览模型和基准目录、检查就绪状态、生成 CLI 命令的最简单方式，无需死记硬背各种参数。

## 启动 TUI

在使用之前，请确保已安装 WorldFoundry 包。推荐使用统一环境脚本：

```bash
bash scripts/setup/unified_install.sh
source tmp/worldfoundry_unified_env.sh
conda activate "${WORLDFOUNDRY_UNIFIED_ENV_PREFIX}"
```

如果是手动管理的 conda 环境，请在代码库根目录安装包含 `[tui]` 依赖组的包：

```bash
pip install -e ".[tui]"
```

环境激活后，启动 TUI：

```bash
worldfoundry-eval tui
```

TUI 读取与 CLI 发现命令相同的 model-zoo 和 benchmark-zoo 清单。它不会绕过清单就绪检查、运行时 profile 或运行器验证。

### 启动选项

| 标志 | 作用 |
| --- | --- |
| `--catalog-json` | 打印 TUI 使用的 catalog payload 后退出。 |
| `--model-id openvla` | 预选一个模型，跳过模型选择器。 |
| `--benchmark-id libero` | 预选一个基准，跳过基准选择器。 |
| `--print-command` | 将生成的 `run` 命令打印到 stdout 并退出，不进入交互式会话。 |

示例——为特定模型 × 基准组合生成命令后退出：

```bash
worldfoundry-eval tui --model-id openvla --benchmark-id libero --print-command
```

这在脚本或 CI 管道中非常有用——你需要标准命令但不需要手动交互。

## 浏览与操作

TUI 是一个四栏终端面板。左侧显示 OpenEnvision 标识、当前 conda 环境，以及可搜索的 **Catalogs** 模型表；右侧是当前模式的概览、操作工具栏、参数表单和 **Execution Log**。

在左上角切换 **Mode**，可在 Inference、Studio、Evaluation 三种模式间切换。三种模式共用同一份 catalog 数据，但提供的控件和生成的命令不同。

### Inference（推理）

Inference 用于选择模型、填写模型相关输入，并生成可执行命令。工具栏提供 **Run**、**Stop**、**Copy**、**Cmd**、**Check**、**Files**、**GPU**、**Sync**。

![TUI Inference 模式](/images/tui/tui-inference.png)

常见字段包括 prompt、输入路径、输出目录、可选 artifact 路径和 GPU id。**Command Preview** 会随参数修改实时更新。

### Studio

Studio 用于从终端启动本地 Studio Web UI。按 **Run** 启动服务，状态和日志会显示在下方执行日志中。

![TUI Studio 模式](/images/tui/tui-studio.png)

### Evaluation（评测）

Evaluation 用于配置 benchmark 运行。选择评测意图、metric 和输出目录后，TUI 会为当前模型 × benchmark 组合生成对应的 `worldfoundry-eval` 命令。

![TUI Evaluation 模式](/images/tui/tui-evaluation.png)

执行日志中会提示快捷键：`Ctrl+Q` 退出、`r` 刷新、`c` 复制、`Tab` / `Shift+Tab` 在面板间切换。

### 参数配置

Evaluation 模式提供两个明确的意图：

- **Model Benchmark** — 选择模型和 benchmark，生成从模型推理到官方 benchmark 的端到端命令。
- **Score Artifact Directory** — 选择 benchmark 和已有 artifact 目录；此时隐藏模型与 metric 子集，因为会执行 benchmark 的完整 protocol。

Inference 模式展示模型相关参数，例如：

- **步数 / 迭代次数** — 去噪或采样步数。
- **引导系数 / cfg** — 无分类器引导 (classifier-free guidance) 强度。
- **序列长度** — 最大 token 或帧数。
- **种子** — 确定性生成种子（可选）。

在 TUI 中更改参数会实时更新生成的命令。需要填写更多 custom dataset manifest 字段时，请使用 Workspace 或 `generate-score` CLI。

### 就绪指示器

TUI 在目录条目旁显示就绪警告：

- **缺失 checkpoint** — 模型要求一个在声明路径中未找到的 checkpoint。
- **缺失数据集** — 基准要求一个本地未暂存的数据集。
- **缺失环境** — 运行时 profile 声明的 conda 包或 CUDA 版本未被满足。
- **License 限制** — 模型或数据集有 gated 或非商业 license，需要手动审查。

当就绪阻碍项存在时，生成的命令默认包含 `--mode official-validation`。这确保你可以验证连接（wiring），而不需要为被阻断的运行消耗 GPU 资源。

## 回退模式

如果未安装 Textual 依赖，TUI 会打印一个静态回退摘要代替交互式面板。回退摘要列出模型 ID、基准 ID 和简短的就绪说明。

要恢复完整的交互式 TUI，安装 `[tui]` 依赖组：

```bash
pip install -e ".[tui]"
```
